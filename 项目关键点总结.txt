  Sparrow_RISC-V 车牌识别项目 — 关键点总结
  芯片: 紫光PDS PG2L100H 6FB676 (盘古676 100Pro+)
  软核: Sparrow RISC-V (小麻雀) @ 27MHz
  输入: HDMI (1920×1080@60Hz) 替代原OV5640摄像头

┌─────────────────────────────────────────────────────────────────────────────┐
│  一、架构总览                                                              │
└─────────────────────────────────────────────────────────────────────────────┘

HDMI输入(MS7200) → fram_buf_raw(DDR) → 图像预处理 → fram_buf_bin(DDR)
                                                            ↓
                                        RISC-V读取二值图 → BFS找车牌区域
                                                            ↓
                                        根据四角坐标裁剪原图 → 透视校正
                                                            ↓
                                        二值化 → 字符分割 → MLP推理识别


┌─────────────────────────────────────────────────────────────────────────────┐
│  二、RTL (Verilog) 关键注意点                                              │
└─────────────────────────────────────────────────────────────────────────────┘

2.1 HDMI输入
    - vsync需取反: .wr_fsync(~vs_in)    (参考例程写法)
    - de直接使用:   .wr_en(de_in)
    - 数据格式:    hdmi_data_in = {r[7:3], g[7:2], b[7:3]}  (RGB565)

2.2 帧冻结控制
    - RISC-V外设4: 地址 0x40000400
    - 写1冻结(停止DDR写入), 写0解除
    - RTL中: fram_buf的.wr_en要gating: de_in & (~frame_freeze)
    - 二值图通路也要gating: proc_de & (~frame_freeze)

2.3 分辨率参数
    - HDMI输入: 1920×1080
    - 二值图有效宽度: 1890 (预处理左右各吞15像素)
    - DDR行步长: 1920像素/行 (DDR_ADDR_OFFSET = 960个32位字)
    - sync_vg时序: 1080p (V_TOTAL=1125, H_TOTAL=2200)

2.4 地址转换 (重要!)
    - sparrow_soc中: axi_araddr = (physical_byte_araddr >> 2)
    - 所以CPU地址要左移2位才能得到实际DDR偏移
    - fram_buf_raw:  ADDR_OFFSET=0x0100_0000 → CPU读 0xA400_0000
    - fram_buf_bin:  ADDR_OFFSET=0x0000_0000 → CPU读 0xA000_0000

2.5 IP核配置
    - PLL: clkout0 = 148.5MHz (1080p像素时钟)
    - wr_fram_buf: CAS_MODE=36K (18K不够)
    - rd_fram_buf: CAS_MODE=18K (读写并发够用)
    - 不能直接用字符串替换改IP核RTL文件 (会破坏不同CAS_MODE的原语参数)

2.6 HDMI RX配置 (关键!)
    - ms7200_ctl.v 中的EQ寄存器值必须匹配参考例程
    - 有13个寄存器的值在不同项目间不同 (0x0D36~0x0D41等)
    - 务必从验证过的例程中复制ms7200_ctl.v

2.7 BOM问题
    - PowerShell的Set-Content / Out-File会添加UTF-8 BOM (EF BB BF)
    - PDS不识别BOM, 会导致Syntax error
    - 需用[System.IO.File]::WriteAllBytes()写入无BOM的文件


┌─────────────────────────────────────────────────────────────────────────────┐
│  三、RISC-V 软核编程注意点                                                 │
└─────────────────────────────────────────────────────────────────────────────┘

3.1 硬件限制
    - 无FPU (浮点运算单元) → 禁止float/double
    - 无硬件除法 → 用软件 div_int() 模拟
    - 有硬件乘法 (RV32M) → 乘法可用

3.2 总线宽度
    - 总线32位 → 读DDR必须用 uint32_t* (volatile)
    - 用uint16_t*会因非对齐访问导致数据错误
    - 两个16位像素打包在一个32位字中: {pixel_odd, pixel_even}
    - 读取: uint32_t w = raw32[addr]; pixel = (w>>shift) & 0xFFFF

3.3 内存限制
    - SRAM: 256KB (数据RAM)
    - IRAM: 256KB (指令RAM, 存储程序代码)
    - 大型数据(映射表/权重)放DDR (地址0xA2000000起)
    - 局部数组不要太大 (< 1KB), 否则栈溢出导致程序重启
    - 需要大缓冲区时用DDR: volatile uint8_t* buf = (volatile uint8_t*)0xA1B00000

3.4 DDR地址映射
    - 二值图:    0xA0000000  (fram_buf_bin, offset 0x00000000)
    - 原图:      0xA4000000  (fram_buf_raw, offset 0x01000000<<2)
    - 校正原图:  0xA1900000  (440×140×2字节)
    - 二值缓存:  0xA1A00000  (440×140/4 字)
    - 投影数组:  0xA1C00000  (水平/垂直)
    - 映射表:    0xA2000000  (440×140×2×4字节 ≈ 493KB)
    - 权重:      0xA2000000  或 SRAM内联
    - MLP测试:   0xA1B00000


┌─────────────────────────────────────────────────────────────────────────────┐
│  四、图像预处理管线 (FPGA硬件实现)                                          │
└─────────────────────────────────────────────────────────────────────────────┘

4.1 处理流程
    fram_buf_raw读 → gaussian5x5(R,G,B) → rgb2gray → sobel_x3x3
      → morph_close_25x3 → binary_frame_out → 二值图(Sobel路径)
    fram_buf_raw读 → rgb_hsv → morph_close_25x3 → 对齐 → 二值图(HSV路径)
    融合: sobel_close_bin & hsv_align_bin → 最终二值图 → fram_buf_bin

4.2 像素损失分析 (每行1920)
    gaussian5x5(5×5):  左右各丢2像素  → 累计: 1916
    sobel_x3x3(3×3):   左右各丢1像素  → 累计: 1914
    morph_close_25x3:  左右各丢12像素 → 累计: 1890
    有效宽度 = 1920 - 30 = 1890


┌─────────────────────────────────────────────────────────────────────────────┐
│  五、BFS连通域分析 (软件实现)                                              │
└─────────────────────────────────────────────────────────────────────────────┘

5.1 原理
    - 扫描二值图每个像素
    - 遇到白色(>0x8000)则开始BFS四方向搜索
    - 用DDR队列存储待访问像素 (MEM_Q_X/Y, 0xA1000000起)
    - 染黑访问过的像素, 避免重复
    - 记录外接矩形 (min_x, max_x, min_y, max_y)

5.2 边界剔除
    - 碰图像边缘的连通域忽略 (桌面边框/噪声)
    - 车牌区域在图像内部, 不会碰到边缘

5.3 寻址公式
    DDR偏移 = y * DDR_STRIDE + x + LEFT_LOSS
    其中 DDR_STRIDE=1920, LEFT_LOSS=15
    BIN_IMG_W=1890 (有效宽度), IMG_H=1080


┌─────────────────────────────────────────────────────────────────────────────┐
│  六、透视校正 (角度矫正)                                                   │
└─────────────────────────────────────────────────────────────────────────────┘

6.1 原理
    - 输入: 车牌四角坐标 (tl, tr, bl, br)
    - 输出: 440×140 正矩形校正图
    - 双线性插值在DDR中预计算映射表

6.2 映射表预计算 (一次, 启动时)
    - 140×440 = 61600 个映射点
    - 每个点存 src_x, src_y (2个int)
    - 共 61600×2×4 = 492,800 字节 ≈ 493KB
    - 放DDR 0xA2000000
    - 每帧直接查表, 不调用除法

6.3 注意事项
    - div_int()非常慢! 不要在像素循环中调用
    - 映射表预计算调用约580次div_int, 可接受
    - 映射表数据放SRAM会溢出(480KB > 256KB)


┌─────────────────────────────────────────────────────────────────────────────┐
│  七、字符识别 (MLP神经网络)                                                │
└─────────────────────────────────────────────────────────────────────────────┘

7.1 模型架构
    - 输入: 16×32 灰度图 (512像素)
    - 隐藏层: 128 → 64 (两个全连接层)
    - 输出: 65类 (0-9, A-Z, 省份简称)
    - 总参数量: 512×128 + 128 + 128×64 + 64 + 64×65 + 65 ≈ 78K int16

7.2 权重量化
    - 训练用sklearn MLPClassifier
    - 导出时量化到 int16 (精度97.1%, 损失仅0.2%)
    - 权重缩放: right-shift 15位 (等价于÷32768)
    - 激活函数: ReLU (max with 0)

7.3 推理代码 (纯整数, 无浮点无除法)
    输入: uint8[512] (0-255)
    h1 = max((input · W1 + B1) >> 15, 0)
    h2 = max((h1 · W2 + B2) >> 15, 0)
    out = (h2 · W3 + B3) >> 15
    argmax(out) → 类别索引

7.4 内存占用
    - int16权重: ~150KB (可放SRAM)
    - 推理中间变量: ~2KB
    - 每个字符推理时间: ~20ms (27MHz)
    - 8个字符: ~0.16秒

7.5 训练数据
    - 来源: GitHub项目 dataset/ann (7z压缩)
    - 65类, 16151张字符图
    - 原始大小20×20, 训练时resize到16×32
    - 训练脚本: export_small2.py → 生成 mlp_small.h


┌─────────────────────────────────────────────────────────────────────────────┤
│  八、常用调试方法                                                           │
└─────────────────────────────────────────────────────────────────────────────┘

8.1 串口发送图像数据
    - 协议: AA55 + type(1byte) + w(2byte) + h(2byte) + pixels + 55AA
    - type=1: 校正后原图(RGB565)
    - type=2: 二值图(8bit/pixel)
    - type=3: 字符图(8bit/pixel, 第3字节为序号)
    - MATLAB接收显示脚本见项目说明

8.2 DDR数据校验
    - 直接printf打印几个关键像素值
    - 注意用32位总线读: volatile uint32_t*
    - 对比HDMI显示画面与DDR数据是否一致

8.3 常见崩溃原因
    - 栈溢出 (局部数组太大) → 改用DDR缓冲区
    - SRAM溢出 (静态数组太大) → 改用DDR
    - 除零 → 确保分母不为0
    - 死循环 (n=10写成n==10) → 检查for循环条件
